# Описание проекта: Aura Scanner

Дата обновления: 18.05.2026

## Что это за проект

`Aura Scanner` — это Flutter-приложение для сканирования, импорта, просмотра, базового редактирования и хранения документов. Проект состоит из мобильного клиента, backend для авторизации и облачной синхронизации, а также отдельного мини-проекта подписи.

Основные части репозитория:

- `scanner_ap` — основное Flutter-приложение.
- `backend` — Node.js/TypeScript backend с MongoDB, JWT и API для документов.
- `signature` — отдельный мини-проект подписи.
- `scripts` — служебные PowerShell-скрипты для анализа и smoke-проверок.

## Что реализовано в `scanner_ap`

### Авторизация

- экран логина (дизайн: карточка с тенью, поля с иконками);
- экран регистрации;
- выход из аккаунта с вызовом `POST /auth/logout` на сервере;
- смена пароля из экрана настроек (`POST /auth/change-password`);
- социальный вход: Google, ВКонтакте, Telegram, Instagram через `flutter_web_auth_2`;
  - Google и VK — полный OAuth flow;
  - Telegram и Instagram — показывают инструкцию (требуют дополнительной настройки);
- автоматическое обновление access token через refresh token (Dio interceptor);
- при истечении access token — прозрачный retry запроса с новым токеном;
- при истечении refresh token — автоматический разлогин.

### Сканирование и захват

- обычное сканирование документов;
- отдельный сценарий для паспорта;
- отдельный сценарий для ID-карты;
- QR-сканер;
- OCR с поддержкой русского языка (Tesseract `rus+eng` как основной, ML Kit Latin как fallback);
- перевод текста через камеру (исправлен баг с зависанием при инициализации);
- экран подписи.

### Работа с документами

- документы сохраняются в `getApplicationDocumentsDirectory()`;
- реестр документов (`DocumentRegistry`) — хранит связку `localPath ↔ remoteId ↔ name` как JSON в SharedPreferences; автоматически мигрирует старый формат;
- просмотр PDF, DOC/DOCX, TXT и изображений;
- открытие файлов через системное приложение (`OpenFilex`);
- превью-миниатюры через `FutureBuilder` (стабильный кэш, без race condition);
- переименование документа (локально + на сервере);
- удаление документа (локально + на сервере);
- шаринг документа через `share_plus`;
- импорт файлов через `file_picker` (с загрузкой на сервер);
- действия с документом через BottomSheet (вместо popup menu).

### Полная облачная синхронизация

`DocumentRegistry` + `DocumentSyncService` обеспечивают двустороннюю синхронизацию:

- **при старте** — показывает локальные файлы сразу, затем в фоне:
  - скачивает документы из облака которых нет локально;
  - повторяет загрузку для файлов у которых нет `remoteId` (прерванный upload);
- **при сохранении** — загружает на сервер в фоне, сохраняет `remoteId`;
- **при удалении** — удаляет локально и `DELETE` на сервере;
- **при переименовании** — переименовывает локально и `PATCH` на сервере;
- **при импорте** — загружает на сервер и регистрирует;
- **оффлайн-режим** — все операции работают локально, синхронизация при следующем запуске.

### Редактирование изображений

- цветокоррекция: яркость, контраст, насыщенность, оттенок;
- тяжёлые операции (`gaussianBlur`, `decodeImage`, `encodePng`) вынесены в изолят через `compute()`;
- дебаунс 350 мс на слайдерах — предотвращает flood compute-вызовов;
- во время обработки показывает старое изображение + полупрозрачный оверлей со спиннером.

### PDF-инструменты

- сжатие PDF;
- извлечение страниц;
- изменение порядка страниц;
- слияние документов;
- чтение информации о PDF через `PdfService`.

### UI и настройки

- экран настроек (`SettingsScreen`): подключение к серверу, готовые адреса, смена пароля, выход;
- экран Premium (`PremiumScreen`): список возможностей, градиентная шапка, карточка с ценой;
- поиск по документам с фильтрацией.

### Тесты

- `test/services/server_config_test.dart` — 8 тестов;
- `test/services/image_editing_service_test.dart` — 11 тестов;
- `test/models/pdf_info_test.dart` — 4 теста;
- `test/widgets/login_screen_test.dart` — 12 тестов.

## Что реализовано в `backend`

### Стек

- Express + TypeScript + MongoDB/Mongoose + JWT + multer
- Winston (структурированные логи) + Morgan (HTTP-логи)
- helmet (security headers) + express-rate-limit + CORS

### API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/health` | Статус сервера + состояние БД |
| POST | `/api/auth/register` | Регистрация |
| POST | `/api/auth/login` | Вход (возвращает `token` + `refreshToken`) |
| POST | `/api/auth/social` | Вход через Google/VK/Telegram/Instagram |
| POST | `/api/auth/refresh` | Обновление access token (ротация refresh token) |
| POST | `/api/auth/logout` | Выход (токен в blacklist, удаление refresh tokens) |
| GET | `/api/auth/profile` | Получить профиль |
| PATCH | `/api/auth/profile` | Обновить имя/email |
| POST | `/api/auth/change-password` | Сменить пароль |
| GET | `/api/documents` | Список документов |
| POST | `/api/documents/upload` | Загрузить файл |
| GET | `/api/documents/:id/download` | Скачать файл |
| PATCH | `/api/documents/:id` | Переименовать |
| DELETE | `/api/documents/:id` | Удалить |

### Безопасность и надёжность

- rate limiting: `/api/auth/*` — 30 req/15 мин; остальное — 200 req/мин;
- JWT blacklist на MongoDB с TTL-индексом (переживает рестарт сервера);
- refresh token с ротацией (MongoDB, TTL 30 дней);
- серверная верификация Google (`tokeninfo` endpoint), VK (API `users.get`), Telegram (HMAC-SHA256);
- ObjectId валидация на всех роутах документов;
- try-catch во всех контроллерах;
- multer ошибки перехватываются (413 при превышении размера);
- email regex + минимальная длина пароля;
- `express.json({ limit: '10mb' })`;
- retry подключения к MongoDB (5 попыток, exponential backoff);
- graceful shutdown по `SIGTERM`/`SIGINT`;
- валидация обязательных env-переменных при старте.

## Что нужно сделать

### Приоритет 1 — Обязательно перед запуском

1. **Скачать tessdata** — `rus.traineddata` и `eng.traineddata` из [tesseract-ocr/tessdata](https://github.com/tesseract-ocr/tessdata) в `scanner_ap/assets/tessdata/`. Без этого OCR кириллицы не работает.

2. **Настроить OAuth** — подставить реальные Client ID в `social_auth_service.dart`:
   - Google: `YOUR_GOOGLE_CLIENT_ID` → получить в [console.cloud.google.com](https://console.cloud.google.com);
   - VK: `YOUR_VK_CLIENT_ID` → получить на [vk.com/editapp](https://vk.com/editapp).

3. **Настроить `.env` на сервере** — сменить `JWT_SECRET`, `JWT_REFRESH_SECRET`, задать `MONGODB_URI`.

4. **`flutter pub get`** — после добавления `flutter_tesseract_ocr` и `flutter_web_auth_2`.

5. **iOS** — добавить в `Info.plist` `CFBundleURLTypes` со схемой `com.example.scanner_ap` для OAuth callback.

### Приоритет 2 — Функциональность

6. **Telegram OAuth** — создать бота через @BotFather, сделать backend HTML-страницу с Telegram Login Widget, раскомментировать `_onTelegramTap()` в `login_screen.dart`.

7. **Instagram OAuth** — добавить server-side обмен кода на token (`POST /api/auth/instagram/exchange`), раскомментировать `_onInstagramTap()`.

8. **Сценарий подписи** — определить продуктовый flow:
   - где хранить сохранённые подписи;
   - как повторно выбирать подпись;

   - как экспортировать отдельно.

9. **Android package name** — сменить с `com.example.scanner_ap` на продуктовый (влияет на OAuth redirect URI).

### Приоритет 3 — Качество

10. **Проверка end-to-end** на реальном Android-устройстве:
    - сканирование → сохранение → синхронизация → просмотр на другом устройстве;
    - регистрация → вход → истечение токена → автоматический refresh → выход.

11. **Redis для token blacklist** — текущий MongoDB-based blacklist корректен, но для высоконагруженного production Redis предпочтительнее.

12. **Редактор содержимого** для PDF/DOCX — сейчас только просмотр, переименование, удаление.

13. **Локализация** — вынести все строки интерфейса в arb-файлы.

14. **Уменьшить дублирование** между модулями документа, паспорта и ID-карты.

15. **Дополнить тесты** — интеграционные тесты синхронизации, тесты для DocumentRegistry.

## Технические ограничения

- Android package name всё ещё шаблонный: `com.example.scanner_ap`.
- Telegram и Instagram OAuth требуют дополнительной серверной части и настройки.
- Token blacklist in-memory сбрасывается при рестарте (заменён на MongoDB, но для production с несколькими инстансами нужен Redis).
- Web и Windows Desktop не подготовлены как целевые платформы.
- Дублирование модуля подписи: папка `signature` и встроенная версия в `scanner_ap`.
