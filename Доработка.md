# Комплексный аудит Aura Scanner

## 🔴 СПРИНТ 1 — КРИТИЧНЫЕ БАГИ (стоппер для релиза)

- [x] **1. Mounted захардкожен в document_utils.dart:32** ✅
  Миксин теперь `on State<T>`, `updateState()` проверяет `mounted`.

- [x] **2. Бесконечный кеш превью в document_utils.dart:26-29 и my_documents_screen.dart:43** ✅
  Внутри миксина `_LruCache` (60 записей), в `MyDocumentsScreen` — `LinkedHashMap` + LRU-выселение (80 записей), очистка в `dispose()`.

- [x] **3. existsSync блокирует UI в my_documents_screen.dart:162** ✅
  Заменено на `await File(...).exists()` в `_downloadAndRegister`. (В `_loadDocuments` и `_syncWithCloud` уже async было.)

- [x] **4. Утечка токенов через URL social_auth_service.dart:64-65** ✅
  Реализован one-time code flow: новая модель `OAuthCode` (TTL 5 мин), endpoint `POST /auth/oauth/exchange`, общий helper `loginOrCreateUserAndIssueCode()`. Google/VK/Telegram callback-и теперь редиректят в `aurascanner://oauth2redirect?code=<code>` — токены в URL не попадают. Для Telegram fragment-кейса добавлен `POST /auth/telegram/exchange`.

- [x] **5. JWT_SECRET по умолчанию `'changeme'` в env.ts:8** ✅
  Дефолт удалён, добавлена жёсткая валидация: минимум 32 символа + чёрный список известных слабых значений (`changeme`, `secret`, и т.д.). Падает при старте — нельзя задеплоить с предсказуемым секретом.

- [x] **6. CORS без origin в app.ts** ✅
  Добавлен whitelist через `CORS_ALLOWED_ORIGINS` (CSV) с dev-дефолтами `localhost:3000/5173`. Запросы без Origin (мобильный клиент) проходят, браузерные с неразрешённым Origin — блокируются с логом.

- [x] **7. Race condition в deep link social_auth_service.dart:51** ✅
  При вызове `waitForLink()` старый `_pendingCompleter` отменяется через `completeError(StateError)` — пользователь увидит явное «Авторизация прервана», старый Future не зависает навсегда. Таймаут уменьшен с 5 до 2 минут.

- [x] **8. RadioGroup<int> в remove_spots_screen.dart:153** ✅
  Оказалось, что `RadioGroup<T>` — это канонический виджет Flutter 3.32+ (старые `groupValue`/`onChanged` на `RadioListTile` deprecated). Сигнатура `onChanged` исправлена: вместо `_isProcessing ? null : ...` (несовместимо с `ValueChanged<int?>`) — проверка `_isProcessing` внутри callback.

## 🟠 СПРИНТ 2 — ВЫСОКИЙ ПРИОРИТЕТ

### Производительность
- [x] **Пагинация в `listDocuments`** ✅
  Контроллер принимает `?limit=` (max 200, default 100) и `?offset=`, ответ теперь `{items, total, limit, offset}` + заголовок `X-Total-Count`. Клиентский `DocumentSyncService.list` поддерживает оба формата (новый + legacy-массив).
- [x] **Debounce на поиске** ✅
  Уже реализован: `Timer? _searchDebounce` + `_onSearchChanged()` запускает `_applySearchFilter` через `Timer(Duration(milliseconds: 250), ...)`, отменяется в `dispose()`. Аудит был неактуален.
- [x] **Синхронный File I/O в сканере** ✅
  В `document_scanner_screen.dart` (`_saveImageToGallery`, `_cropImage`) заменено `existsSync()` на `await exists()`, добавлен `if (!mounted) return;` перед `ScaffoldMessenger`. `_deleteDocument` в `my_documents_screen.dart` обёрнул `File.delete()` в fire-and-forget async-замыкание. (Остальные `readAsBytes()` уже были async — false-positive аудита.)
- [x] **PDF preview** ✅
  Перепроверил `document_utils.dart` — `_renderFirstPdfPageToImage` рендерит один проход в нужном размере; дубля 2x нет, аудит был ошибочным.
- [x] **Timeout в AI-сервисе** ✅
  `_send()` в `ai_service.dart` теперь использует `Options(sendTimeout: 60s, receiveTimeout: 90s)`, в catch отлавливается `DioExceptionType.sendTimeout/receiveTimeout/connectionTimeout` → юзер-френдли «AI не отвечает».

### Заглушки / недореализовано
- [x] **4 кнопки в камере** ✅
  `camera.dart:_openToolFeature` теперь маршрутизирует:
  - `Icons.highlight` → `HighlightScreen`
  - `Icons.vpn_key_outlined` → `AddPasswordScreen`
  - `Icons.delete_forever_outlined` → `RemoveWatermarkScreen`
  - `Icons.eco` → `DocumentAiScreen.eco()` (использует существующий `AIService().analyzeEcoPackaging`).
- [x] **PDF viewer `_saveDocument()`** ✅
  Вместо фейкового снэкбара — реальное сохранение: viewer обёрнут в `RepaintBoundary`, при «Сохранить» захватывается PNG (pixelRatio 2.0) и записывается в `getApplicationDocumentsDirectory()` как `<имя>_annotated_<timestamp>.png`. В снэкбаре путь к файлу, ошибки тоже показываются.
- [x] **DOCX превью с реальным контентом** ✅
  Текст и так извлекался через `docxToText(bytes)`, но рендер 120×80 px со шрифтом 4 pt делал результат нечитаемой кашей — отсюда впечатление «фейка». Переписал `_createRealDocxPreviewImage`: размер 160×220 (как страница), первая непустая строка как заголовок (10 pt, w700) с разделителем, остальной текст как тело (7 pt, до 100 строк), `canvas.clipRect` чтобы не вылезало за рамку. Теперь видно, что за документ.
- [x] **VK OAuth ручка** ✅
  Зарегистрирована: `GET /api/auth/vk/login` (в `auth.routes.ts`) и корневой `GET /vk_id_redirect` (в `app.ts`, для совпадения с Universal Link / associated domain). Callback использует `loginOrCreateUserAndIssueCode()` — токены не утекают в URL.

### Безопасность
- [x] **Rate limit на /auth/social** ✅
  Добавлен `strictAuthLimiter` (10 запросов / 5 минут) поверх общего, применён на `/login`, `/register`, `/social`, `/oauth/exchange`, `/telegram/exchange`.
- [x] **N+1 в refresh token verify** ✅
  Заменены два round-trip'а на один с `RefreshToken.findOne(...).populate<{userId: {...}}>('userId', '_id email name')`.
- [x] **Premium auto-deactivate** ✅
  Новый `backend/src/utils/premium.ts`: `isPremiumActive(user)` (реактивная проверка `premiumExpiresAt > now` на чтении в `getProfile`/`updateProfile`) + фоновый `startPremiumSweeper()` (запуск сразу + каждый час, `setInterval.unref()`). Профиль возвращает `premiumExpiresAt` в ответе.
- [x] **RevokedToken TTL** ✅
  Проверил `models/RevokedToken.ts` — TTL-индекс `{ expiresAt: 1 }` с `expireAfterSeconds: 0` уже на месте. Mongo сама подчищает.

### UX
- [x] **Переименование голосовых записей** ✅
  В `voice_note_screen.dart` добавлен `_renameNote(_VoiceNote)`: AlertDialog с TextField (autofocus, prefilled basename без расширения), санитизация запрещённых символов `[\\/:*?"<>|]` → `_`, защита от коллизий имён, остановка плеера если активен, `File.rename()` + обновление списка и `SharedPreferences`. UI: иконка `Icons.drive_file_rename_outline` рядом с кнопкой удаления. (trim/edit — out of scope, отложено.)
- [x] **Preview в merge при добавлении файлов** ✅
  Метод `_buildLeading(path, ext)` в `merge_documents_screen.dart`: для JPG/PNG/JPEG — `Image.file` 44×44 в `ClipRRect`, для PDF — рендер первой страницы через `PdfDocument.openFile` + `compute(_bgraToJpegIsolate, ...)`, кешируется в `Map<String, Future<Uint8List?>> _pdfThumbCache` (запись удаляется при `removeAt`). Спиннер пока грузится, fallback-иконка при ошибке.
- [x] **Progress callback при сжатии/слиянии PDF** ✅
  `PdfService.compressPdf` принимает `onProgress(current, total)` — вызывается перед началом и после каждой страницы. В `compress_pdf_screen` диалог через `ValueListenableBuilder<({int current, int total})>` показывает «Сжатие: 5/12 стр.» + детерминированный `LinearProgressIndicator`. - [x] **_isBusy overlay в remote_documents с подписью операции** ✅
  Добавлено поле `String? _busyLabel` и хелпер `_setBusy(label)`; все вызовы `setState(() => _isBusy = ...)` заменены на `_setBusy('Загрузка в облако…' | 'Скачивание…' | 'Переименование…' | 'Удаление…' | null)`. ### Технический долг
- [x] **Memory leak в voice_note_screen** ✅
  `_loadNotes()` после фильтрации несуществующих путей перезаписывает `SharedPreferences` очищенным списком — мёртвые пути больше не накапливаются между запусками.
- [x] **Async без await в camera.dart `_launchInBrowser`** ✅
  Сигнатура изменена с `void … async` на `Future<void> … async`. Caller (`_onQrViewCreated`) явно оборачивает в `unawaited(...)` чтобы fire-and-forget был осознанный, а не случайный.
- [x] **BGRA→JPEG конверсия off main thread** ✅
  В `merge_documents_screen.dart` вынес `_bgraToJpegIsolate` в top-level функцию с payload-классом `_BgraPayload(pixels, width, height)`. Вызов в `_addPdfPages` теперь `await compute(_bgraToJpegIsolate, ...)` — `image.encodeJpg` каждой страницы PDF исполняется в отдельном изоляте, главный поток не блокируется и UI/progress остаются отзывчивыми при объединении многостраничных PDF.
- [x] **Таймаут deep_link_service** ✅
  Уже уменьшён в Sprint 1 c 5 минут до 2-х (`waitForLink({Duration timeout = const Duration(minutes: 2)})`) — фишинг-окно сокращено.
